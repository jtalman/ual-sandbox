Package  SXF


The  package SXF implements the front end for  the Standard eXchange Format .  It is based on the lex/yacc tools and provides the framework for producing output in a target language or data structures.  The extensibility has been the main criterion of the SXF design and its implementation approach.  The SXF can be configured in several ways to support different project requirements: introducing new element types (e.g.  a RHIC  helical dipole or a CESR wiggler ),  specializing bucket of element attributes (e.g. entry or exit thin multipoles vs. COSY INFINITY fringe fields), and including new element buckets common  for all element types.  The configuration mechanism  is built on  several  extensible registries, such as NodeRegistry and ElemBucketRegistry, and the effective hash mapping technique, such as a perfect hash function generator (GPERF).
  • sxf.ll  -  FLEX  lexical patterns.
  • sxf.yy - YACC grammar .
  • element.gperf - GPERF keyfile for the NodeRegistry.
